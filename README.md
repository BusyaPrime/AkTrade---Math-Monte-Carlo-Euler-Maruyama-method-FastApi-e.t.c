# AkTrade

Продвинутый торговый терминал и аналитическая платформа с бэкендом на FastAPI и фронтендом на Next.js. 

Проект разбит как монорепа, чтобы строго отделить UI от математического ядра.

> **ДИСКЛЕЙМЕР**: AkTrade создан в первую очередь как демонстрация архитектурных, математических и full-stack компетенций для портфолио. Это исследовательский пет-проект, иллюстрирующий, как можно элегантно связывать современные frontend-фреймворки со сложной количественной аналитикой (Python/numpy) и криптографией прямо в браузере. Вызовы к биржам и торговым движкам являются заглушками (mock), но вся математика и шифрование под капотом абсолютно реальны.

## Стек

* **Frontend**: Next.js 14-16 (App Router), React 19, TypeScript. Дизайн-система на Tailwind CSS v4 + Radix UI (shadcn). Графики крутятся на `lightweight-charts` (отрисовка японских свечей на Canvas) и `recharts` (SVG-графики для аналитики).
* **Backend**: FastAPI (Python 3.10+), Uvicorn для ASGI, `numpy` (для векторизации матана), WebSockets.
* **База**: PostgreSQL 18, Redis 7 (легко поднимаются через Docker). Схемы моделей описаны через Prisma ORM (`schema.prisma` лежит внутри веб-воркспейса).

## Под капотом (Ключевые фичи)

### 🌍 Локализация, Таймзоны и Доступность (a11y)
* **i18n (EN/RU)**: Мультиязычность из коробки реализована через `next-intl`. Сделан гибкий middleware, который маршрутизирует юзеров по префиксу `/[locale]/`. Словари лежат отдельно в `messages/`.
* **Таймзоны**: Все торговые графики (Lightweight Charts) и аналитические дашборды жестко привязаны к локальному времени браузера юзера (или времени, выбранному в настройках профиля). Никакого захардкоженного UTC — полная синхронизация с часовым поясом трейдера.
* **Темизация и Colorblind Mode**: Кроме базового переключателя Dark/Light через `next-themes`, палитра Tailwind спроектирована адаптивно. Цвета `destructive` (красный для шортов/убытков) и `success` (зеленый для лонгов) можно подменять через CSS-переменные на сине-оранжевую палитру для режима дальтоников без переписывания компонентов. Деградированная SVG-шумность (glassmorphism) тоже отключается для слабых девайсов.

### 🔒 Сквозное шифрование (E2EE Чат)
Встроенный мессенджер работает по принципу Zero-Knowledge — сервер FastAPI используется исключительно как транспортный реле и не может расшифровать переписку.
1. **Генерация ключей**: При загрузке вкладки Web Crypto API генерирует асимметричную пару ключей на эллиптической кривой **P-256**. Приватный ключ никогда не уходит в сеть и живет только в RAM браузере.
2. **Обмен (ECDH)**: Клиенты скачивают открытые ключи друг друга и через алгоритм Diffie-Hellman получают идентичный симметричный ключ (`Shared Secret`).
3. **Общение (AES-GCM)**: Сообщения шифруются AES-GCM с генерацией уникального 12-байтного вектора инициализации (IV) для каждого пакета. Сервер хранит и пересылает только Base64-строки.

### 🧮 Квантовое ядро выш мата: Прыжковая Диффузия и Симуляции Монте-Карло
Большинство дефолтных аналитических систем (и пет-проектов) считают риски по классическому Геометрическому Броуновскому Движению (GBM). Фундаментальная слабость GBM — допущение о нормальном распределении доходностей и константной волатильности. В реальном мире (особенно в крипте и высокотехе) рынки не живут в вакууме: они страдают от кластеризации волатильности, флеш-крашей и "жирных хвостов" (fat tails).

Поэтому движок оценки рисков под капотом AkTrade реализует **прыжково-диффузионную модель Бейтса (Bates Jump-Diffusion Model)**. Это институциональное расширение модели Хестона, которое решает главные проблемы неэффективных рынков:

1. **Стохастическая волатильность (Heston Process)**: Волатильность оторвана от цены и представляет собой полностью независимый стохастический процесс с возвратом к среднему (mean-reverting drift). Мы используем **разложение Холецкого (Cholesky decomposition)** для внедрения эффекта левериджа (p < 0). Математика настроена так, что как только спотовая цена летит в пропасть, ковариационная матрица автоматически и нелинейно задирает волатильность алгоритма. Модель математически "понимает" наступление паники.
2. **Пуассоновские скачки (Poisson Jumps)**: Графики реальных активов прерывисты. Твиты инфлюенсеров или макро-дампы маркетмейкеров ломают лог-нормальное распределение. Чтобы алгоритм учитывал появление "черных лебедей", в стохастическое дифференциальное уравнение встроен скачкообразный процесс Пуассона (lambda dt). Именно он формирует экстремальный эксцесс в левом хвосте распределения кривой.

**Вычислительный Пайплайн:**
Бэкенд не гоняет медленные циклы. Интегратор Эйлера-Маруямы векторизован на тензорах `numpy`. Движок генерирует N=5000 независимых траекторий Монте-Карло, просчитывая сотни тысяч дифференциальных шагов буквально за миллисекунды аппаратного времени. 

Сведя тысячи вариантов будущего в единое распределение, алгоритм отсекает крайний левый сектор и формирует бескомпромиссные метрики риск-менеджмента:
* **VaR 95% (Value at Risk)**: Доверительный порог. Точка отсечения, показывающая предел убытков в 95% сценариев.
* **CVaR 95% (Expected Shortfall / Условный VaR)**: Истинная цена риска. Математическое ожидание потерь в тех самых 5% катастрофических исходов, когда пробивается VaR и начинается каскад маржин-коллов.

## 🛠 Локальный запуск (Development)

Проект легко поднимается по частям.

### 1. Инфраструктура (Postgres + Redis)
Сначала нужно поднять БД.
```bash
cd infra/docker
docker-compose up -d
```
*Контейнеры повиснут на дефолтных портах (5432 и 6379).*

### 2. Бэкенд (FastAPI)
```bash
cd apps/api
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install fastapi uvicorn httpx numpy

uvicorn app.main:app --reload --port 8000
```
*Бэк заведется на http://localhost:8000, поднимет REST-роуты и WebSocket-хаб для котировок.*

### 3. Фронтенд (Next.js)
```bash
cd apps/web
npm install
npm run dev
```
*Рендерится на http://localhost:3003. NextAuth настроен на `jwt` токены в куках, Prisma-адаптер на этапе разработки умышленно погашен для ускорения, так что войти в дашборд можно под любыми кредами (заглушка CredentialsProvider).*
